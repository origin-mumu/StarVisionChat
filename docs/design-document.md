### 1.3 用户故事详情

#### US-01：摄像头画面描述
**场景**：用户打开应用，授权摄像头权限后，AI 自动开始分析画面内容。

**交互流程**：
1. 用户打开页面，点击"开始"按钮
2. 浏览器弹出摄像头权限请求
3. 用户授权后，摄像头画面显示在页面上
4. AI 每隔 1-2 秒分析一帧画面
5. AI 以文字+语音的方式描述画面内容

**验收标准**：
- [ ] 能正确调用摄像头
- [ ] 画面分析延迟 < 3 秒
- [ ] 能识别常见物体、人物、场景

---

#### US-02：语音对话
**场景**：用户对着麦克风说话，AI 听懂后以语音回复。

**交互流程**：
1. 用户按住"说话"按钮（或自动检测语音）
2. 用户说出问题
3. 松开按钮后，语音被识别为文字
4. AI 生成回复并以语音播报

**验收标准**：
- [ ] 能正确调用麦克风
- [ ] 中文识别准确率 > 90%
- [ ] 语音合成自然流畅

---

#### US-03：视觉问答
**场景**：用户指着画面中的某个物体问"这是什么？"

**交互流程**：
1. 用户对着摄像头展示一个物体
2. 用户问"这是什么？"或"帮我看看这个"
3. AI 结合当前画面和语音进行理解
4. AI 给出针对性的回答

**验收标准**：
- [ ] 能正确理解指代关系
- [ ] 回答与画面内容相关

---

#### US-04：语音播报
**场景**：AI 的回复以语音形式播报，用户无需阅读文字。

**交互流程**：
1. AI 生成文字回复
2. 同时调用 TTS 合成语音
3. 语音通过浏览器播放
4. 页面同步显示文字

**验收标准**：
- [ ] 语音合成延迟 < 1 秒
- [ ] 语音播放流畅无卡顿

---

#### US-05：语音打断
**场景**：AI 正在播报时，用户想问新问题。

**交互流程**：
1. AI 正在播放语音回复
2. 用户开始说话
3. 系统检测到用户语音，立即停止 AI 播报
4. 开始处理用户的新问题

**验收标准**：
- [ ] 打断响应时间 < 500ms
- [ ] 不会丢失用户语音

---

#### US-06：多轮对话
**场景**：用户进行连续对话，AI 能记住上下文。

**交互流程**：
1. 用户："这是什么？" → AI："这是一个苹果"
2. 用户："它是什么颜色的？" → AI："它是红色的"
3. AI 能理解"它"指的是上一轮提到的苹果

**验收标准**：
- [ ] 保留最近 10 轮对话上下文
- [ ] 能正确进行指代消解

---

#### US-08：状态提示
**场景**：用户能看到 AI 当前的处理状态。

**交互流程**：
1. 用户说话时显示"🎤 正在聆听..."
2. 语音识别时显示"✏️ 正在识别..."
3. AI 思考时显示"🤔 正在思考..."
4. AI 回复时显示"💬 正在回复..."

**验收标准**：
- [ ] 状态切换及时准确
- [ ] 视觉反馈清晰

## 二、成本控制策略

### 2.1 我想到的成本控制技巧

| # | 技巧 | 原理 | 预期节省 |
|---|------|------|---------|
| C-01 | **帧采样** | 不每帧都发送，仅 1-2 FPS | ~90% 视觉 API |
| C-02 | **场景变化检测** | 画面不变不分析 | ~50-70% 额外 |
| C-03 | **本地 VAD** | 前端检测语音，静音不发送 | ~30% STT |
| C-04 | **本地 Whisper** | STT 完全本地化 | 100% STT |
| C-05 | **Edge-TTS** | 使用免费 TTS 服务 | 100% TTS |
| C-06 | **模型分级** | 简单问题用 mini，复杂用强 | ~60-80% |
| C-07 | **Prompt 压缩** | 精简系统提示词 | ~20% token |
| C-08 | **上下文摘要** | 长对话压缩而非全量 | ~30% token |
| C-09 | **响应缓存** | 相似问题缓存结果 | ~10-20% |
| C-10 | **图像压缩** | 降低分辨率和质量 | ~50% 带宽 |
| C-11 | **批量请求** | 合并短时间内的多帧 | ~20% API 调用 |
| C-12 | **智能降级** | 网络差时自动降级到轻量模型 | - |

## 三、技术决策记录

### 3.1 为什么选择 FastAPI？
- 原生异步支持，适合 I/O 密集型任务
- WebSocket 支持完善
- 类型提示 + 自动文档
- Python 生态丰富（Whisper、PIL、ffmpeg 等）

### 3.2 为什么选择本地 Whisper？
- 免费，无 API 调用费用
- 中文识别质量高
- 可离线使用
- 延迟可控（base 模型约 1 秒）

### 3.3 为什么选择 Edge-TTS？
- 完全免费
- 音质尚可
- 支持中文
- 无需 API Key

### 3.4 前端为什么不用 React/Vue？
- 减少构建复杂度
- 直接调用 Web API 更灵活
- 项目规模适合原生 JS
- 降低学习成本
