# StarVisionChat - AI 视觉对话助手 技术需求文档

## 一、项目概述

### 1.1 项目名称
StarVisionChat - AI 视觉对话助手

### 1.2 项目目标
开发一款基于 Web 的 AI 视觉对话应用，用户打开摄像头和麦克风后，AI 能够实时"看到"摄像头画面、"听到"用户语音，并以语音+文字的方式给予自然流畅的回应。

### 1.3 核心价值
- **视觉理解**：AI 能准确识别和理解摄像头中的画面内容
- **语音交互**：支持自然的语音输入和语音输出，降低交互门槛
- **成本可控**：通过端云协同策略，在保证体验的前提下控制运营成本

### 2.2 技术栈选型

| 层级 | 技术选型 | 选型理由 |
|------|---------|---------|
| **前端** | Vue 3 + Vite + Composition API | 组件化开发，响应式状态管理，Vite 构建快，生态成熟 |
| **前端 UI** | Element Plus / Naive UI | 成熟的 Vue 3 UI 组件库，快速搭建界面 |
| **前端通信** | WebSocket (原生 API) | 直接调用浏览器 WebSocket API，无需额外封装 |
| **后端框架** | FastAPI (Python) | 异步高性能，原生支持 WebSocket，类型提示完善 |
| **后端 AI SDK** | openai SDK + httpx | 直接调用 API，无需 LangChain 等重型框架（详见下方说明） |
| **实时通信** | WebSocket | 全双工通信，低延迟音视频流传输 |
| **语音识别 (STT)** | OpenAI Whisper (本地) / Whisper API (云端) | 本地 Whisper 免费且质量高，云端作为降级方案 |
| **语音合成 (TTS)** | Edge-TTS (免费) / OpenAI TTS (高质量) | Edge-TTS 免费无限制，OpenAI TTS 音质更好 |
| **视觉理解** | GPT-4o / Claude Vision / 本地 LLaVA | 多模型可选，按成本和精度需求切换 |
| **对话 LLM** | GPT-4o-mini / Claude Haiku / 本地 Qwen | 轻量模型处理简单对话，复杂场景升级模型 |
| **音频处理** | ffmpeg + pydub | 成熟的音视频处理库 |

#### 2.2.1 关于 LangChain 的说明

**本项目不使用 LangChain**，原因如下：

| 考虑点 | 分析 |
|--------|------|
| **项目需求** | 主要是调用 Vision API + STT + TTS，链路简单直接 |
| **LangChain 优势** | 复杂 Chain、Agent、RAG、多工具编排 — 本项目用不到 |
| **LangChain 劣势** | 增加依赖、调试困难、抽象层开销、版本更新频繁 |
| **推荐方案** | 直接用 `openai` SDK + `httpx` 调用 API，简单高效 |

对话记忆管理使用自实现的简单滑动窗口 + 摘要压缩策略，无需引入 LangChain 的 Memory 模块。

## 三、功能需求

### 3.1 核心功能模块

#### 模块 1：摄像头采集与视觉处理
- **F1.1** 支持调用用户摄像头，获取实时视频流
- **F1.2** 前端进行帧采样（非每帧都发送），降低带宽和计算成本
- **F1.3** 支持场景变化检测，仅在画面有显著变化时触发 AI 分析
- **F1.4** 图像压缩与分辨率自适应，根据网络状况动态调整

#### 模块 2：语音交互
- **F2.1** 支持调用用户麦克风，获取实时音频流
- **F2.2** 前端 VAD（Voice Activity Detection）检测，区分语音和静音
- **F2.3** 语音转文字（STT）：支持中英文混合识别
- **F2.4** 文字转语音（TTS）：AI 回复以语音播报
- **F2.5** 支持打断（Barge-in）：用户说话时自动停止 AI 播报

#### 模块 3：AI 对话
- **F3.1** 多轮对话上下文管理（保留最近 N 轮）
- **F3.2** 视觉+语音多模态融合理解
- **F3.3** 流式响应：AI 边生成边播报，降低感知延迟
- **F3.4** 支持追问和指代消解（"这个是什么？"结合画面理解）

#### 模块 4：成本控制
- **F4.1** 智能帧采样，减少不必要的视觉 API 调用
- **F4.2** 模型分级：简单问题用轻量模型，复杂问题用强模型
- **F4.3** 本地优先策略：STT/TTS 优先使用本地模型
- **F4.4** 成本监控面板，实时显示 API 调用次数和费用

## 四、技术实现方案

### 4.1 前端实现

#### 4.1.1 摄像头采集
```javascript
// 核心逻辑：通过 MediaStream API 获取摄像头
// 1. getUserMedia() 获取视频流
// 2. Canvas 定时截帧（1-2 FPS，非实时全帧）
// 3. 压缩为 JPEG (quality=0.6) 后发送
// 4. 场景变化检测：对比相邻帧差异，仅变化时发送
```

#### 4.1.2 麦克风采集
```javascript
// 核心逻辑：AudioWorklet 实现低延迟音频采集
// 1. getUserMedia({audio: true}) 获取音频流
// 2. 前端 VAD 检测语音段
// 3. 语音段缓冲后通过 WebSocket 发送
// 4. 支持静音检测自动结束
```

### 4.2 后端实现

#### 4.2.1 WebSocket 通信协议
```python
# 消息类型定义
class MessageType:
    # 客户端 -> 服务端
    VIDEO_FRAME = "video_frame"       # 视频帧（base64 JPEG）
    AUDIO_CHUNK = "audio_chunk"       # 音频片段
    TEXT_INPUT = "text_input"         # 文本输入（调试用）
    
    # 服务端 -> 客户端
    AI_RESPONSE = "ai_response"       # AI 文字回复
    AI_AUDIO = "ai_audio"            # AI 语音回复
    STATUS = "status"                 # 状态更新
    ERROR = "error"                   # 错误信息
```

#### 4.2.2 视觉处理流水线
```python
# 视觉处理流程
# 1. 接收 base64 图像
# 2. 场景变化检测（可选，与前端双重校验）
# 3. 图像预处理（裁剪、增强）
# 4. 调用视觉模型 API
# 5. 返回结构化描述
```

#### 4.2.3 语音处理流水线
```python
# STT 流程
# 1. 接收音频 chunk
# 2. 音频预处理（降噪、归一化）
# 3. 语音活动检测（服务端 VAD）
# 4. 拼接完整语音段
# 5. 调用 Whisper 识别
# 6. 返回文字结果

# TTS 流程
# 1. 接收文字
# 2. 流式调用 TTS API
# 3. 分片返回音频
# 4. 前端边接收边播放
```

#### 4.2.4 对话管理
```python
# 会话管理策略
# - 每个 WebSocket 连接 = 一个会话
# - 维护最近 10 轮对话上下文
# - 视觉描述作为系统上下文注入
# - 支持上下文窗口滑动和摘要压缩
```

## 五、成本控制策略

### 5.1 端侧优化（减少云端调用）

| 策略 | 预期节省 | 实现难度 |
|------|---------|---------|
| **帧采样**：仅 1-2 FPS，非每帧发送 | ~90% 视觉 API 调用 | 低 |
| **场景变化检测**：画面不变不分析 | ~50-70% 额外节省 | 中 |
| **本地 VAD**：前端检测语音，静音不发送 | ~30% STT 调用 | 中 |
| **本地 STT**：Whisper 本地运行 | 100% STT 费用 | 中 |
| **本地 TTS**：Edge-TTS 免费 | 100% TTS 费用 | 低 |

### 5.2 云端优化（降低单次成本）

| 策略 | 预期节省 | 实现难度 |
|------|---------|---------|
| **模型分级**：简单问题用 mini 模型 | ~60-80% 模型费用 | 中 |
| **Prompt 优化**：精简系统提示词 | ~20% token 费用 | 低 |
| **响应缓存**：相似问题缓存结果 | ~10-20% API 调用 | 中 |
| **流式响应**：减少用户等待，间接降低重试 | - | 低 |
| **上下文压缩**：长对话摘要而非全量 | ~30% token 费用 | 中 |

### 5.3 成本预估（每用户每小时）

| 场景 | 无优化 | 优化后 | 节省比 |
|------|-------|-------|-------|
| 视觉 API | $18 (30FPS) | $0.5 (1FPS+变化检测) | 97% |
| STT | $3.6 | $0 (本地) | 100% |
| TTS | $5.4 | $0 (Edge-TTS) | 100% |
| LLM 对话 | $2 | $0.8 (分级) | 60% |
| **总计** | **$29/小时** | **$1.3/小时** | **95%** |

## 六、非功能需求

### 6.1 性能指标
- **首帧响应**：< 3 秒（从用户说话到 AI 开始回复）
- **语音识别延迟**：< 1.5 秒（本地 Whisper）
- **视觉分析延迟**：< 2 秒（云端 API）
- **端到端延迟**：< 4 秒（完整一轮对话）
- **并发支持**：单实例支持 50+ WebSocket 连接

### 6.2 兼容性
- **浏览器**：Chrome 90+, Edge 90+, Firefox 100+, Safari 16+
- **操作系统**：Windows 10+, macOS 12+, Ubuntu 20.04+
- **设备**：需要摄像头和麦克风（或外接设备）

### 6.3 安全性
- 音视频数据不落盘，处理后即丢弃
- API Key 仅在服务端，不暴露给前端
- WebSocket 连接支持 token 认证
- 支持 HTTPS/WSS 加密传输

## 八、风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| 摄像头权限被拒绝 | 无法使用 | 提供文本输入作为降级方案 |
